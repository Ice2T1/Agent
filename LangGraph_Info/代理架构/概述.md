# LangGraph 代理架构

## 什么是代理？

**代理**是使用 LLM 来决定应用程序控制流的系统。与硬编码固定控制流的工作流不同，代理能自主控制如何完成任务。

## 代理 vs 工作流

| 工作流 | 代理 |
|--------|------|
| 预定义代码路径编排 LLM 和工具 | LLM 动态指导流程和工具使用 |
| 确定性执行 | 自主决策 |
| 适合固定流程 | 适合复杂、动态任务 |

## LLM 控制应用程序的方式

1. **路由决策**：在两条潜在路径之间选择
2. **工具选择**：决定调用众多工具中的哪一个
3. **终止判断**：决定答案是否足够或需要更多工作

## 代理类型

### 1. 路由器（Router）

LLM 从一组特定选项中选择一个步骤。控制级别有限，专注于单个决策。

#### 实现方式

**结构化输出**

```python
from pydantic import BaseModel, Field
from typing import Literal

class RouteDecision(BaseModel):
    step: Literal["search", "calculate", "answer"] = Field(
        description="Next step to take"
    )
    reasoning: str = Field(description="Why this step was chosen")

# 使用结构化输出
router_llm = llm.with_structured_output(RouteDecision)
decision = router_llm.invoke("How far is the moon?")

print(f"Next step: {decision.step}")
print(f"Reasoning: {decision.reasoning}")
```

**工具调用**

```python
from langchain_core.tools import tool

@tool
def search(query: str) -> str:
    """Search for information"""
    return search_api(query)

@tool
def calculate(expression: str) -> float:
    """Calculate mathematical expression"""
    return eval(expression)

# LLM 决定调用哪个工具
llm_with_tools = llm.bind_tools([search, calculate])
response = llm_with_tools.invoke("What is 2 + 2?")

if response.tool_calls:
    tool_to_call = response.tool_calls[0]
    result = tools[tool_to_call["name"]].invoke(tool_to_call["args"])
```

### 2. 工具调用代理（ReAct 模式）

比路由器更复杂，支持：
- **多步决策**：连续做出一系列决策
- **工具访问**：选择并使用各种工具

#### ReAct 架构三要素

**1. 工具调用**
```python
tools = [search_tool, calculator_tool, database_tool]
llm_with_tools = llm.bind_tools(tools)
```

**2. 记忆**
```python
from langgraph.graph import MessagesState

class AgentState(MessagesState):
    # 消息历史自动维护
    pass
```

**3. 规划**
```python
# 在 while 循环中重复调用
while not done:
    response = llm_with_tools.invoke(messages)
    
    if response.tool_calls:
        # 执行工具
        results = execute_tools(response.tool_calls)
        messages.append(response)
        messages.extend(results)
    else:
        # 最终答案
        done = True
        final_answer = response.content
```

#### 使用预构建代理

```python
from langgraph.prebuilt import create_react_agent

agent = create_react_agent(
    model=llm,
    tools=[search_tool, calculator_tool],
    prompt="You are a helpful assistant."
)

result = agent.invoke({
    "messages": [{"role": "user", "content": "What is the population of Paris?"}]
})
```

### 3. 自定义代理架构

针对特定任务定制代理架构通常能带来更好性能。

**反思**
```python
def reflect_and_improve(state: State):
    output = state["current_output"]
    
    # 评估输出质量
    evaluation = evaluator_llm.invoke(f"Evaluate: {output}")
```

## 常见代理架构模式

### 1. 提示链（Prompt Chaining）

任务分解为一系列固定子任务，每个 LLM 调用处理前一个调用的输出。

```python
from langgraph.graph import StateGraph, START, END

class State(TypedDict):
    topic: str
    outline: str
    draft: str
    final: str

def generate_outline(state: State):
    outline = llm.invoke(f"Create outline for: {state['topic']}")
    return {"outline": outline}

def write_draft(state: State):
    draft = llm.invoke(f"Write draft based on: {state['outline']}")
    return {"draft": draft}

def polish(state: State):
    final = llm.invoke(f"Polish: {state['draft']}")
    return {"final": final}

# 线性工作流
builder = StateGraph(State)
builder.add_node("outline", generate_outline)
builder.add_node("draft", write_draft)
builder.add_node("polish", polish)
builder.add_edge(START, "outline")
builder.add_edge("outline", "draft")
builder.add_edge("draft", "polish")
builder.add_edge("polish", END)

chain = builder.compile()
```

**适用场景**：
- 任务可清晰分解为固定子任务
- 牺牲延迟换取准确性

### 2. 并行化（Parallelization）

LLM 同时处理任务，编程方式聚合输出。

**变体**：
- **分段**：任务分解为独立子任务并行运行
- **投票**：多次运行同一任务获取多样化输出

```python
class State(TypedDict):
    topic: str
    joke: str
    story: str
    poem: str
    combined: str

def generate_joke(state: State): return {"joke": llm.invoke(f"Joke about {state['topic']}")}
def generate_story(state: State): return {"story": llm.invoke(f"Story about {state['topic']}")}
def generate_poem(state: State): return {"poem": llm.invoke(f"Poem about {state['topic']}")}

def aggregate(state: State):
    combined = f"Joke: {state['joke']}\nStory: {state['story']}\nPoem: {state['poem']}"
    return {"combined": combined}

builder = StateGraph(State)
builder.add_node("joke", generate_joke)
builder.add_node("story", generate_story)
builder.add_node("poem", generate_poem)
builder.add_node("aggregate", aggregate)

# 并行执行
builder.add_edge(START, "joke")
builder.add_edge(START, "story")
builder.add_edge(START, "poem")
builder.add_edge("joke", "aggregate")
builder.add_edge("story", "aggregate")
builder.add_edge("poem", "aggregate")
builder.add_edge("aggregate", END)
```

**适用场景**：
- 子任务可并行化提高速度
- 需要多个视角提高置信度
- 复杂任务分解为独立方面

### 3. 路由（Routing）

对输入进行分类并导向专门后续任务。

```python
class RouteDecision(BaseModel):
    category: Literal["technical", "creative", "factual"]

router = llm.with_structured_output(RouteDecision)

class State(TypedDict):
    input: str
    category: str
    output: str

def route_input(state: State):
    decision = router.invoke(state["input"])
    return {"category": decision.category}

def technical_response(state: State):
    return {"output": llm.invoke(f"Technical: {state['input']}")}

def creative_response(state: State):
    return {"output": llm.invoke(f"Creative: {state['input']}")}

def factual_response(state: State):
    return {"output": llm.invoke(f"Factual: {state['input']}")}

builder = StateGraph(State)
builder.add_node("route", route_input)
builder.add_node("technical", technical_response)
builder.add_node("creative", creative_response)
builder.add_node("factual", factual_response)

builder.add_edge(START, "route")
builder.add_conditional_edges(
    "route",
    lambda s: s["category"],
    {
        "technical": "technical",
        "creative": "creative",
        "factual": "factual"
    }
)
builder.add_edge("technical", END)
builder.add_edge("creative", END)
builder.add_edge("factual", END)
```

**适用场景**：
- 具有不同类别的复杂任务
- 各类别最好单独处理
- 可通过 LLM 准确分类

### 4. 编排器 - 工作器（Orchestrator-Worker）

中央 LLM 动态分解任务，委托给工作器 LLM，并综合结果。

```python
from typing import List, Annotated
import operator

class Section(BaseModel):
    name: str
    description: str

class Sections(BaseModel):
    sections: List[Section]

planner = llm.with_structured_output(Sections)

class State(TypedDict):
    topic: str
    sections: List[Section]
    completed_sections: Annotated[list, operator.add]
    final_report: str

def orchestrator(state: State):
    # 生成计划
    plan = planner.invoke([
        {"role": "system", "content": "Generate report sections"},
        {"role": "user", "content": f"Topic: {state['topic']}"}
    ])
    return {"sections": plan.sections}

def worker(state: State):
    # 为每个 section 创建 worker
    section = state["section"]
    content = llm.invoke(f"Write section: {section.name} - {section.description}")
    return {"completed_sections": [content]}

def assign_workers(state: State):
    # 动态创建 worker
    return [Send("worker", {"section": s}) for s in state["sections"]]

def synthesizer(state: State):
    report = "\n\n".join(state["completed_sections"])
    return {"final_report": report}

builder = StateGraph(State)
builder.add_node("orchestrator", orchestrator)
builder.add_node("worker", worker)
builder.add_node("synthesizer", synthesizer)

builder.add_edge(START, "orchestrator")
builder.add_conditional_edges("orchestrator", assign_workers, ["worker"])
builder.add_edge("worker", "synthesizer")
builder.add_edge("synthesizer", END)
```

**适用场景**：
- 无法预测所需子任务的复杂任务
- 子任务不是预定义的
- 需要灵活分解任务

### 5. 评估器 - 优化器（Evaluator-Optimizer）

一个 LLM 生成响应，另一个在循环中提供评估和反馈。

```python
class Feedback(BaseModel):
    grade: Literal["pass", "fail"]
    feedback: str

evaluator = llm.with_structured_output(Feedback)

class State(TypedDict):
    topic: str
    draft: str
    feedback: str
    grade: str
    final: str

def generator(state: State):
    if state.get("feedback"):
        prompt = f"Improve based on feedback: {state['feedback']}\n\nPrevious: {state['draft']}"
    else:
        prompt = f"Generate: {state['topic']}"
    
    draft = llm.invoke(prompt)
    return {"draft": draft}

def evaluator_node(state: State):
    feedback = evaluator.invoke([
        {"role": "system", "content": "Evaluate the output"},
        {"role": "user", "content": state["draft"]}
    ])
    return {"grade": feedback.grade, "feedback": feedback.feedback}

def should_continue(state: State):
    if state["grade"] == "pass":
        return "accept"
    else:
        return "revise"

builder = StateGraph(State)
builder.add_node("generator", generator)
builder.add_node("evaluator", evaluator_node)

builder.add_edge(START, "generator")
builder.add_edge("generator", "evaluator")
builder.add_conditional_edges(
    "evaluator",
    should_continue,
    {"accept": END, "revise": "generator"}
)
```

**适用场景**：
- 有明确的评估标准
- 迭代细化能提供可衡量价值
- LLM 能提供有意义的反馈

## 记忆（Memory）

### 短期记忆
- 序列中早期步骤的信息
- 通过消息历史维护

### 长期记忆
- 跨会话信息
- 使用 Store 接口

```python
from langgraph.store.memory import InMemoryStore

store = InMemoryStore()

def remember(state: State, store: BaseStore):
    # 保存记忆
    store.put(("user_memories",), "memory_1", {"fact": "User likes Python"})
    
    # 检索记忆
    memories = store.search(("user_memories",))
    
    return {"context": memories}
```

## 工具调用

工具允许代理与外部系统交互（API、数据库等）。

```python
from langchain_core.tools import tool

@tool
def book_flight(from_airport: str, to_airport: str) -> str:
    """Book a flight"""
    return f"Booked flight from {from_airport} to {to_airport}"

@tool
def book_hotel(hotel_name: str) -> str:
    """Book a hotel"""
    return f"Booked stay at {hotel_name}"

# 绑定到 LLM
llm_with_tools = llm.bind_tools([book_flight, book_hotel])

# LLM 决定调用哪个工具
response = llm_with_tools.invoke("Book a flight from BOS to JFK")

if response.tool_calls:
    tool_call = response.tool_calls[0]
    result = tools[tool_call["name"]].invoke(tool_call["args"])
```

## 选择代理架构

| 架构 | 复杂度 | 自主性 | 适用场景 |
|------|--------|--------|----------|
| 路由器 | 低 | 低 | 简单分类、路由 |
| 提示链 | 低 | 无 | 固定流程任务 |
| 并行化 | 中 | 无 | 多视角、批量处理 |
| 工具调用 | 中 | 中 | 多步骤问题解决 |
| 编排器 - 工作器 | 高 | 高 | 复杂、动态任务分解 |
| 评估器 - 优化器 | 高 | 中 | 需要质量控制的场景 |

## 最佳实践

1. **从简单开始**：先尝试简单架构，必要时增加复杂度
2. **明确评估标准**：特别是评估器 - 优化器模式
3. **提供足够上下文**：帮助 LLM 做出更好决策
4. **设置合理边界**：限制最大迭代次数、工具调用次数
5. **监控和调试**：使用 LangSmith 等工具追踪代理行为
6. **人机协作**：关键决策点加入人工审查
