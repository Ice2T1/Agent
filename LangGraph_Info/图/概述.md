# LangGraph 图与运行时完整指南

## 第一部分：图核心概念

### 核心组件

LangGraph 将代理工作流建模为图，包含三个关键组件：

#### 1. State（状态）
- 共享数据结构，表示应用程序的当前快照
- 可以是任何 Python 类型，通常是 `TypedDict` 或 `Pydantic BaseModel`
- 所有 Nodes 和 Edges 的输入模式

#### 2. Nodes（节点）
- 编码代理逻辑的 Python 函数（同步或异步）
- 接收当前 State 作为输入，执行计算或副作用，返回更新的 State
- 可以包含 LLM 或普通 Python 代码
- 在幕后被转换为 `RunnableLambda`，支持批处理和异步

#### 3. Edges（边）
- 根据当前 State 决定接下来执行哪个 Node
- 可以是条件分支或固定转换

**核心原则**：节点完成工作，边指示下一步该做什么。

### 执行模型

LangGraph 受 Google Pregel 系统启发，使用消息传递和超步（sup-step）机制：

- **超步**：图节点的一次迭代
  - 并行运行的节点属于同一个超步
  - 顺序运行的节点属于不同的超步
- **执行流程**：
  1. 节点在初始时处于 inactive 状态
  2. 当节点在传入边上接收到新消息时变为 active
  3. 活跃节点运行函数并返回更新
  4. 没有传入消息的节点投票停止（标记为 inactive）
  5. 当所有节点都 inactive 且无消息传输时，执行终止

### StateGraph API

#### 编译图

```python
graph = graph_builder.compile(...)
```

编译步骤：
- 对图结构进行基本检查（如没有孤立节点）
- 可指定运行时参数（检查点、断点等）
- **必须先编译图才能使用**

#### 状态定义

**使用 TypedDict（推荐）**

```python
from typing_extensions import TypedDict

class State(TypedDict):
    foo: int
    bar: list[str]
```

**使用 Pydantic BaseModel**

```python
from pydantic import BaseModel

class State(BaseModel):
    foo: int
    bar: list[str]
```

**显式输入/输出模式**

当需要不同的输入和输出模式时：

```python
class InputState(TypedDict):
    user_input: str

class OutputState(TypedDict):
    graph_output: str

class OverallState(TypedDict):
    foo: str
    user_input: str
    graph_output: str

class PrivateState(TypedDict):
    bar: str

def node_1(state: InputState) -> OverallState:
    return {"foo": state["user_input"] + " name"}

def node_2(state: OverallState) -> PrivateState:
    return {"bar": state["foo"] + " is"}

def node_3(state: PrivateState) -> OutputState:
    return {"graph_output": state["bar"] + " Lance"}

builder = StateGraph(OverallState, input=InputState, output=OutputState)
builder.add_node("node_1", node_1)
builder.add_node("node_2", node_2)
builder.add_node("node_3", node_3)
builder.add_edge(START, "node_1")
builder.add_edge("node_1", "node_2")
builder.add_edge("node_2", "node_3")
builder.add_edge("node_3", END)

graph = builder.compile()
result = graph.invoke({"user_input": "My"})
# {'graph_output': 'My name is Lance'}
```

**关键点**：
- 节点可以写入图状态中的任何通道，即使不在输入模式中
- 节点可以声明额外的状态通道，只要模式已定义

### 归约器（Reducers）

归约器决定节点更新如何应用到 State。State 中每个键都有独立的归约函数。

#### 默认归约器（覆盖）

未指定归约函数时，更新会覆盖该键：

```python
class State(TypedDict):
    foo: int
    bar: list[str]

# 输入：{"foo": 1, "bar": ["hi"]}
# Node 1 返回：{"foo": 2}
# 结果：{"foo": 2, "bar": ["hi"]}

# Node 2 返回：{"bar": ["bye"]}
# 结果：{"foo": 2, "bar": ["bye"]}  # 覆盖
```

#### 使用 Annotated 指定归约器

```python
from typing import Annotated
from operator import add

class State(TypedDict):
    foo: int
    bar: Annotated[list[str], add]

# 输入：{"foo": 1, "bar": ["hi"]}
# Node 1 返回：{"foo": 2}
# 结果：{"foo": 2, "bar": ["hi"]}

# Node 2 返回：{"bar": ["bye"]}
# 结果：{"foo": 2, "bar": ["hi", "bye"]}  # 累加
```

#### 消息归约器

使用消息列表时，推荐使用 `add_messages`：

```python
from langchain_core.messages import AnyMessage
from langgraph.graph.message import add_messages
from typing import Annotated
from typing_extensions import TypedDict

class GraphState(TypedDict):
    messages: Annotated[list[AnyMessage], add_messages]
```

`add_messages` 的优势：
- 跟踪消息 ID，更新时覆盖现有消息
- 自动反序列化为 LangChain Message 对象
- 支持以下格式：
  ```python
  {"messages": [HumanMessage(content="message")]}
  {"messages": [{"type": "human", "content": "message"}]}
  ```

#### MessagesState（预构建）

```python
from langgraph.graph import MessagesState

class State(MessagesState):
    documents: list[str]
```

### 节点（Nodes）

#### 基本用法

```python
from typing_extensions import TypedDict
from langchain_core.runnables import RunnableConfig
from langgraph.graph import StateGraph

class State(TypedDict):
    input: str
    results: str

def my_node(state: State, config: RunnableConfig):
    print("In node: ", config["configurable"]["user_id"])
    return {"results": f"Hello, {state['input']}!"}

def my_other_node(state: State):
    return state

builder = StateGraph(State)
builder.add_node("my_node", my_node)
builder.add_node("other_node", my_other_node)
```

#### 无名称节点

```python
builder.add_node(my_node)
# 可通过 "my_node" 引用
```

#### 节点缓存

```python
from langgraph.cache.memory import InMemoryCache
from langgraph.types import CachePolicy
import time

class State(TypedDict):
    x: int
    result: int

def expensive_node(state: State) -> dict[str, int]:
    time.sleep(2)
    return {"result": state["x"] * 2}

builder = StateGraph(State)
builder.add_node("expensive_node", expensive_node, cache_policy=CachePolicy(ttl=3))
builder.set_entry_point("expensive_node")
builder.set_finish_point("expensive_node")

graph = builder.compile(cache=InMemoryCache())

# 第一次调用
print(graph.invoke({"x": 5}, stream_mode='updates'))  
# [{'expensive_node': {'result': 10}}]

# 第二次调用（缓存）
print(graph.invoke({"x": 5}, stream_mode='updates'))  
# [{'expensive_node': {'result': 10}, '__metadata__': {'cached': True}}]
```

### 边（Edges）

#### 普通边

```python
graph.add_edge("node_a", "node_b")
```

#### 条件边

```python
def routing_function(state):
    # 返回下一个节点的名称
    if state["foo"] == "bar":
        return "node_b"
    return "node_c"

graph.add_conditional_edges("node_a", routing_function)
```

使用映射字典：

```python
graph.add_conditional_edges("node_a", routing_function, {True: "node_b", False: "node_c"})
```

#### 入口点（START 节点）

```python
from langgraph.graph import START

graph.add_edge(START, "node_a")
```

#### 条件入口点

```python
graph.add_conditional_edges(START, routing_function)
# 或带映射
graph.add_conditional_edges(START, routing_function, {True: "node_b", False: "node_c"})
```

#### END 节点

```python
from langgraph.graph import END

graph.add_edge("node_a", END)
```

### Send API（动态边）

用于 map-reduce 模式，动态创建边：

```python
from langgraph.constants import Send

def continue_to_jokes(state: OverallState):
    return [Send("generate_joke", {"subject": s}) for s in state['subjects']]

graph.add_conditional_edges("node_a", continue_to_jokes)
```

### Command（控制流 + 状态更新）

结合状态更新和路由：

```python
def my_node(state: State) -> Command[Literal["my_other_node"]]:
    return Command(
        update={"foo": "bar"},
        goto="my_other_node"
    )
```

动态控制流：

```python
def my_node(state: State) -> Command[Literal["my_other_node"]]:
    if state["foo"] == "bar":
        return Command(update={"foo": "baz"}, goto="my_other_node")
```

**重要**：必须添加返回类型注解，包含节点路由到的节点名称列表。

#### 导航到父图（子图场景）

```python
def my_node(state: State) -> Command[Literal["other_subgraph"]]:
    return Command(
        update={"foo": "bar"},
        goto="other_subgraph",
        graph=Command.PARENT  # 导航到父图
    )
```

#### 在工具中使用

```python
from langchain_core.tools import tool

@tool
def transfer_to_bob(state) -> Command:
    return Command(
        goto="bob",
        update={"my_state_key": "my_state_value"},
        graph=Command.PARENT,
    )
```

使用 `create_react_agent` / `ToolNode` 时：

```python
def call_tools(state):
    commands = [tools_by_name[tool_call["name"]].invoke(tool_call) for tool_call in tool_calls]
    return commands
```

### 人工参与

使用 `interrupt()` 收集用户输入：

```python
def human_node(state):
    user_input = interrupt(value="Ready for user input")
    return Command(resume=user_input)
```

### 图迁移

LangGraph 支持图定义的迁移：
- 对于未中断的线程，可以更改图的整个拓扑结构（节点、边的增删改）

## 第二部分：运行时（Pregel）详解

### 概述

Pregel 是 LangGraph 的运行时实现，管理 LangGraph 应用程序的执行。编译 `StateGraph` 或创建 `entrypoint` 会生成一个 Pregel 实例。

> **命名来源**：Google 的 Pregel 算法，一种使用图进行大规模并行计算的方法。

### 执行模型

Pregel 将执行器（actors）和通道（channels）组合成一个应用程序。执行器从通道读取数据并向通道写入数据。

执行按照**批量同步并行**（Bulk Synchronous Parallel）模型组织成多个步骤。

#### 执行步骤的三个阶段

每个步骤包括：

**1. 规划（Plan）**
- 确定在此步骤中要执行哪些执行器
- 第一步：选择订阅特殊输入通道的执行器
- 后续步骤：选择订阅前一步骤中更新的通道的执行器

**2. 执行（Execute）**
- 并行执行所有选定的执行器
- 直到全部完成、一个失败或达到超时
- 在此阶段，通道更新对执行器不可见，直到下一个步骤

**3. 更新（Update）**
- 用此步骤中执行器写入的值更新通道

重复此过程，直到没有执行器被选中执行，或达到最大步骤数。

### 核心组件

#### 执行器（PregelNode）

- 订阅通道，从它们读取数据，并向它们写入数据
- 实现了 LangChain 的 `Runnable` 接口
- 可被认为是 Pregel 算法中的一个执行器

#### 通道（Channels）

通道用于在执行器之间进行通信。每个通道都有：
- 值类型
- 更新类型
- 更新函数（接受一系列更新并修改存储的值）

**内置通道类型**

**LastValue**（默认）
- 存储发送到通道的最后一个值
- 适用于输入和输出值
- 或将数据从一个步骤发送到下一个步骤

**Topic**
- 可配置的 PubSub 主题
- 适用于在执行器之间发送多个值
- 或用于累积输出
- 可配置为重复数据删除或在多个步骤中累积值

**BinaryOperatorAggregate**
- 存储一个持久值
- 通过对当前值和发送到通道的每个更新应用二元运算符进行更新
- 适用于在多个步骤中计算聚合

示例：
```python
total = BinaryOperatorAggregate(int, operator.add)
```

### 直接使用 Pregel 示例

#### 单节点示例

```python
from langgraph.channels import EphemeralValue
from langgraph.pregel import Pregel, Channel 

node1 = (
    Channel.subscribe_to("a")
    | (lambda x: x + x)
    | Channel.write_to("b")
)

app = Pregel(
    nodes={"node1": node1},
    channels={
        "a": EphemeralValue(str),
        "b": EphemeralValue(str),
    },
    input_channels=["a"],
    output_channels=["b"],
)

result = app.invoke({"a": "foo"})
# {'b': 'foofoo'}
```

### 高级 API

LangGraph 提供两种高级 API 创建 Pregel 应用程序：

#### 1. StateGraph（Graph API）

更高级别的抽象，简化 Pregel 应用程序的创建：

```python
from typing import TypedDict, Optional
from langgraph.constants import START
from langgraph.graph import StateGraph

class Essay(TypedDict):
    topic: str
    content: Optional[str]
    score: Optional[float]

def write_essay(essay: Essay):
    return {"content": f"Essay about {essay['topic']}"}

def score_essay(essay: Essay):
    return {"score": 10}

builder = StateGraph(Essay)
builder.add_node(write_essay)
builder.add_node(score_essay)
builder.add_edge(START, "write_essay")

# 编译图会返回 Pregel 实例
graph = builder.compile()
```

**检查节点和通道**

编译后的 Pregel 实例与节点和通道列表相关联：

```python
print(graph.nodes)
# {'__start__': <langgraph.pregel.read.PregelNode at 0x...>,
#  'write_essay': <langgraph.pregel.read.PregelNode at 0x...>,
#  'score_essay': <langgraph.pregel.read.PregelNode at 0x...>}

print(graph.channels)
# {'topic': <langgraph.channels.last_value.LastValue at 0x...>,
#  'content': <langgraph.channels.last_value.LastValue at 0x...>,
#  'score': <langgraph.channels.last_value.LastValue at 0x...>,
#  '__start__': <langgraph.channels.ephemeral_value.EphemeralValue at 0x...>,
#  'write_essay': <langgraph.channels.ephemeral_value.EphemeralValue at 0x...>,
#  'score_essay': <langgraph.channels.ephemeral_value.EphemeralValue at 0x...>,
#  'branch:__start__:__self__:write_essay': <langgraph.channels.ephemeral_value.EphemeralValue at 0x...>,
#  ...}
```

#### 2. 函数式 API

（详见函数式 API 文档）

### 关键概念对比

| 概念 | Graph API | Pregel 底层 |
|------|-----------|-------------|
| 节点 | `add_node()` | `PregelNode` |
| 状态 | `State` TypedDict | `Channels` |
| 边 | `add_edge()` | 通道连接 |
| 编译 | `.compile()` | `Pregel` 实例 |

### 使用建议

- **大多数用户**：使用 StateGraph API 或函数式 API，无需直接与 Pregel 交互
- **高级场景**：需要精细控制执行时，可直接使用 Pregel API
- **调试/理解**：了解 Pregel 有助于理解图的执行机制

## 快速参考

### 完整示例：构建一个简单的图

```python
from typing import TypedDict, Annotated, Literal
from operator import add
from langgraph.graph import StateGraph, START, END
from langgraph.types import Command

# 定义状态
class State(TypedDict):
    messages: Annotated[list, add]
    data: str

# 定义节点
def node_a(state: State) -> Command[Literal["node_b", "node_c"]]:
    print("In node A")
    if state["data"] == "go_b":
        return Command(goto="node_b", update={"messages": ["A processed"]})
    return Command(goto="node_c", update={"messages": ["A processed"]})

def node_b(state: State):
    print("In node B")
    return {"messages": ["B processed"], "data": "done"}

def node_c(state: State):
    print("In node C")
    return {"messages": ["C processed"], "data": "done"}

# 构建图
builder = StateGraph(State)
builder.add_node("node_a", node_a)
builder.add_node("node_b", node_b)
builder.add_node("node_c", node_c)
builder.add_edge(START, "node_a")
builder.add_edge("node_b", END)
builder.add_edge("node_c", END)

# 编译
graph = builder.compile()

# 使用
result = graph.invoke({"messages": [], "data": "go_b"})
print(result)
```

### 常见模式

**线性工作流**
```python
builder.add_edge(START, "node_1")
builder.add_edge("node_1", "node_2")
builder.add_edge("node_2", END)
```

**条件路由**
```python
def router(state):
    if condition:
        return "path_a"
    return "path_b"

builder.add_conditional_edges("decision", router)
```

**并行执行**
```python
builder.add_edge(START, "node_a")
builder.add_edge(START, "node_b")
builder.add_edge("node_a", "aggregate")
builder.add_edge("node_b", "aggregate")
```

**Map-Reduce**
```python
def expand(state):
    return [Send("worker", {"item": item}) for item in state["items"]]

builder.add_conditional_edges("start", expand)
builder.add_edge("worker", "reduce")
```
