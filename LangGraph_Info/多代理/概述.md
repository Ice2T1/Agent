# LangGraph 多代理系统

## 什么是多代理系统？

随着代理系统复杂度增加，单个代理可能面临：
- 工具过多导致决策困难
- 上下文过于复杂难以跟踪
- 需要多个专业领域

**解决方案**：将应用分解为更小、独立的代理，组合成多代理系统。

## 多代理系统的优势

### 1. 模块化
- 分离的代理使开发、测试和维护更容易
- 各代理可独立开发和优化

### 2. 专业化
- 创建专注于特定领域的专家代理
- 提高整个系统的性能

### 3. 控制
- 明确控制代理如何通信
- 不依赖函数调用进行协调

## 多代理架构模式

### 1. 网络（Network）

每个代理与所有其他代理通信（多对多），任何代理可决定下一步调用哪个代理。

```python
from typing import Literal
from langchain_openai import ChatOpenAI
from langgraph.types import Command
from langgraph.graph import StateGraph, MessagesState, START, END

model = ChatOpenAI()

def agent_1(state: MessagesState) -> Command[Literal["agent_2", "agent_3", END]]:
    # LLM 决定下一个代理
    response = model.invoke([
        {"role": "system", "content": "Decide which agent should handle this next"},
        *state["messages"]
    ])
    
    # 假设 response 包含 next_agent 字段
    next_agent = extract_next_agent(response)  # 'agent_2' / 'agent_3' / '__end__'
    
    return Command(
        goto=next_agent,
        update={"messages": [response]}
    )

def agent_2(state: MessagesState) -> Command[Literal["agent_1", "agent_3", END]]:
    response = model.invoke(state["messages"])
    next_agent = extract_next_agent(response)
    return Command(goto=next_agent, update={"messages": [response]})

def agent_3(state: MessagesState) -> Command[Literal["agent_1", "agent_2", END]]:
    response = model.invoke(state["messages"])
    next_agent = extract_next_agent(response)
    return Command(goto=next_agent, update={"messages": [response]})

# 构建网络
builder = StateGraph(MessagesState)
builder.add_node(agent_1)
builder.add_node(agent_2)
builder.add_node(agent_3)
builder.add_edge(START, "agent_1")

network = builder.compile()
```

**适用场景**：
- 没有明确代理层次结构
- 没有特定代理调用顺序
- 代理间平等协作

### 2. 主管（Supervisor）

添加一个主管节点（LLM）决定接下来应该调用哪个代理节点。

```python
from typing import Literal
from langchain_openai import ChatOpenAI
from langgraph.types import Command
from langgraph.graph import StateGraph, MessagesState, START, END

model = ChatOpenAI()

def supervisor(state: MessagesState) -> Command[Literal["agent_1", "agent_2", END]]:
    # 主管决定下一个代理
    response = model.invoke([
        {"role": "system", "content": "You are a supervisor. Decide which agent should handle this."},
        *state["messages"]
    ])
    
    next_agent = extract_next_agent(response)
    return Command(goto=next_agent)

def agent_1(state: MessagesState) -> Command[Literal["supervisor"]]:
    response = model.invoke(state["messages"])
    return Command(
        goto="supervisor",
        update={"messages": [response]}
    )

def agent_2(state: MessagesState) -> Command[Literal["supervisor"]]:
    response = model.invoke(state["messages"])
    return Command(
        goto="supervisor",
        update={"messages": [response]}
    )

# 构建主管架构
builder = StateGraph(MessagesState)
builder.add_node(supervisor)
builder.add_node(agent_1)
builder.add_node(agent_2)
builder.add_edge(START, "supervisor")

supervisor_graph = builder.compile()
```

**适用场景**：
- 有明确的控制中心
- 需要协调多个专业代理
- 适合并行运行多个代理

### 3. 主管（工具调用）

子代理作为工具暴露给主管，主管使用工具调用 LLM 决定调用哪个代理。

```python
from typing import Annotated
from langchain_openai import ChatOpenAI
from langgraph.prebuilt import create_react_agent, InjectedState

model = ChatOpenAI()

# 子代理作为工具
def agent_1(state: Annotated[dict, InjectedState]):
    response = model.invoke(state["messages"])
    return response.content  # 返回字符串，自动转为 ToolMessage

def agent_2(state: Annotated[dict, InjectedState]):
    response = model.invoke(state["messages"])
    return response.content

# 主管使用工具调用
tools = [agent_1, agent_2]
supervisor = create_react_agent(model, tools)
```

**适用场景**：
- 主管代理使用工具调用 LLM
- 子代理作为专用工具
- 简化主管逻辑

### 4. 分层（Hierarchical）

创建由单个主管管理的独立代理团队，以及管理这些团队的顶层主管。

```python
from typing import Literal
from langchain_openai import ChatOpenAI
from langgraph.graph import StateGraph, MessagesState, START, END
from langgraph.types import Command

model = ChatOpenAI()

# === 团队 1 ===
class Team1State(MessagesState):
    next: Literal["team_1_agent_1", "team_1_agent_2", "__end__"]

def team_1_supervisor(state: Team1State) -> Command[Literal["team_1_agent_1", "team_1_agent_2", END]]:
    response = model.invoke(state["messages"])
    next_agent = extract_next_agent(response)
    return Command(goto=next_agent)

def team_1_agent_1(state: Team1State) -> Command[Literal["team_1_supervisor"]]:
    response = model.invoke(state["messages"])
    return Command(goto="team_1_supervisor", update={"messages": [response]})

def team_1_agent_2(state: Team1State) -> Command[Literal["team_1_supervisor"]]:
    response = model.invoke(state["messages"])
    return Command(goto="team_1_supervisor", update={"messages": [response]})

# 构建团队 1
team_1_builder = StateGraph(Team1State)
team_1_builder.add_node(team_1_supervisor)
team_1_builder.add_node(team_1_agent_1)
team_1_builder.add_node(team_1_agent_2)
team_1_builder.add_edge(START, "team_1_supervisor")
team_1_graph = team_1_builder.compile()

# === 团队 2 ===
class Team2State(MessagesState):
    next: Literal["team_2_agent_1", "team_2_agent_2", "__end__"]

def team_2_supervisor(state: Team2State):
    # 类似团队 1
    ...

team_2_graph = team_2_builder.compile()

# === 顶层主管 ===
def top_level_supervisor(state: MessagesState) -> Command[Literal["team_1_graph", "team_2_graph", END]]:
    response = model.invoke([
        {"role": "system", "content": "Decide which team should handle this"},
        *state["messages"]
    ])
    next_team = extract_next_team(response)  # 'team_1_graph' / 'team_2_graph'
    return Command(goto=next_team)

# 构建顶层图
builder = StateGraph(MessagesState)
builder.add_node(top_level_supervisor)
builder.add_node("team_1_graph", team_1_graph)
builder.add_node("team_2_graph", team_2_graph)
builder.add_edge(START, "top_level_supervisor")
builder.add_edge("team_1_graph", "top_level_supervisor")
builder.add_edge("team_2_graph", "top_level_supervisor")

graph = builder.compile()
```

**适用场景**：
- 主管难以管理所有代理
- 需要分层组织
- 代理团队有明确分工

### 5. 自定义工作流

提前在自定义工作流中定义代理的调用顺序。

## 交接（Handoffs）

交接是一个代理将控制权移交给另一个代理的模式。

### 创建交接工具

```python
from typing import Annotated
from langchain_core.tools import tool, InjectedToolCallId
from langgraph.prebuilt import InjectedState
from langgraph.types import Command

def create_handoff_tool(*, agent_name: str, description: str = None):
    name = f"transfer_to_{agent_name}"
    description = description or f"Transfer to {agent_name}"
    
    @tool(name, description=description)
    def handoff_tool(
        state: Annotated[MessagesState, InjectedState],
        tool_call_id: Annotated[str, InjectedToolCallId],
    ) -> Command:
        # 创建工具消息确认交接
        tool_message = {
            "role": "tool",
            "content": f"Successfully transferred to {agent_name}",
            "name": name,
            "tool_call_id": tool_call_id,
        }
        
        return Command(
            goto=agent_name,
            update={"messages": state["messages"] + [tool_message]},
            graph=Command.PARENT,  # 导航到父图中的代理
        )
    
    return handoff_tool
```

### 使用交接构建多代理系统

```python
from langgraph.prebuilt import create_react_agent
from langgraph.graph import StateGraph, MessagesState, START

# 创建交接工具
transfer_to_hotel = create_handoff_tool(
    agent_name="hotel_assistant",
    description="Transfer to hotel booking assistant"
)
transfer_to_flight = create_handoff_tool(
    agent_name="flight_assistant",
    description="Transfer to flight booking assistant"
)

# 定义代理（使用预构建的 ReAct 代理）
flight_assistant = create_react_agent(
    model="anthropic:claude-3-5-sonnet-latest",
    tools=[book_flight_tool, transfer_to_hotel],
    prompt="You are a flight booking assistant",
    name="flight_assistant"
)

hotel_assistant = create_react_agent(
    model="anthropic:claude-3-5-sonnet-latest",
    tools=[book_hotel_tool, transfer_to_flight],
    prompt="You are a hotel booking assistant",
    name="hotel_assistant"
)

# 构建多代理图
multi_agent_graph = (
    StateGraph(MessagesState)
    .add_node(flight_assistant)
    .add_node(hotel_assistant)
    .add_edge(START, "flight_assistant")
    .compile()
)

# 使用
result = multi_agent_graph.stream(
    {"messages": [{"role": "user", "content": "Book a flight and hotel"}]},
    subgraphs=True
)
```

### 控制代理输入

使用 `Send()` 直接向工作代理发送数据：

```python
from langgraph.types import Send

def create_task_handoff_tool(*, agent_name: str):
    @tool
    def handoff_tool(
        task_description: Annotated[
            str,
            "Description of what the next agent should do"
        ],
        state: Annotated[MessagesState, InjectedState],
    ) -> Command:
        task_message = {"role": "user", "content": task_description}
        agent_input = {**state, "messages": [task_message]}
        
        return Command(
            goto=[Send(agent_name, agent_input)],
            graph=Command.PARENT,
        )
    
    return handoff_tool
```

## 通信与状态管理

### 消息传递方式

#### 1. 共享完整思维过程

代理共享完整的消息历史（"草稿本"）。

```python
class SharedState(MessagesState):
    # 所有代理共享 messages
    pass

def agent_a(state: SharedState):
    # 可以看到之前所有代理的完整对话
    full_history = state["messages"]
    response = llm.invoke(full_history)
    return {"messages": [response]}
```

**优点**：
- 帮助其他代理做出更好决策
- 提高系统推理能力

**缺点**：
- "草稿本"迅速增长
- 可能需要额外内存管理

#### 2. 只共享最终结果

代理拥有私有"草稿本"，只共享最终结果。

```python
def agent_a(state: MessagesState) -> Command[Literal["agent_b"]]:
    # 内部多轮对话（私有）
    internal_messages = [...]
    internal_response = llm.invoke(internal_messages)
    
    # 只返回最终结果
    return Command(
        goto="agent_b",
        update={"messages": [internal_response]}  # 只共享最终结果
    )
```

**优点**：
- 适用于复杂代理系统
- 减少消息历史长度

**缺点**：
- 丢失中间推理过程
- 需要不同状态模式

## 多轮对话

```python
from langgraph.types import interrupt, Command

class MultiAgentState(MessagesState):
    last_active_agent: str

def human_node(state: MultiAgentState) -> Command[Literal["agent_a", "agent_b"]]:
    """收集用户输入并路由到活跃代理"""
    user_input = interrupt(value="Ready for user input")
    
    # 路由到最后一个活跃的代理
    active_agent = state["last_active_agent"]
    
    return Command(
        update={
            "messages": [{"role": "user", "content": user_input}]
        },
        goto=active_agent
    )

def agent_a(state: MultiAgentState) -> Command[Literal["agent_b", "human"]]:
    response = agent_a_llm.invoke(state["messages"])
    
    if needs_specialist(response):
        return Command(
            goto="agent_b",
            update={
                "messages": [response],
                "last_active_agent": "agent_a"
            }
        )
    else:
        return Command(
            goto="human",
            update={
                "messages": [response],
                "last_active_agent": "agent_a"
            }
        )

# 构建图
builder = StateGraph(MultiAgentState)
builder.add_node("agent_a", agent_a)
builder.add_node("agent_b", agent_b)
builder.add_node("human", human_node)
builder.add_edge(START, "agent_a")

graph = builder.compile(checkpointer=checkpointer)

# 多轮对话
config = {"configurable": {"thread_id": "1"}}

# 第 1 轮
graph.invoke({"messages": [{"role": "user", "content": "I need help"}]}, config)

# 第 2 轮（恢复）
graph.invoke(Command(resume="Book a flight"), config)

# 第 3 轮
graph.invoke(Command(resume="I prefer morning flights"), config)
```

## 完整示例：旅行预订系统

```python
from langchain_anthropic import ChatAnthropic
from langgraph.graph import MessagesState, StateGraph, START
from langgraph.prebuilt import create_react_agent, InjectedState
from langgraph.types import Command, interrupt
from langgraph.checkpoint.memory import MemorySaver
from typing import Annotated

model = ChatAnthropic(model="claude-3-5-sonnet-latest")

class MultiAgentState(MessagesState):
    last_active_agent: str

# 创建交接工具
def create_handoff_tool(*, agent_name: str):
    @tool
    def handoff_tool(
        state: Annotated[MessagesState, InjectedState],
        tool_call_id: Annotated[str, InjectedToolCallId],
    ) -> Command:
        tool_message = {
            "role": "tool",
            "content": f"Transferred to {agent_name}",
            "name": f"transfer_to_{agent_name}",
            "tool_call_id": tool_call_id,
        }
        return Command(
            goto=agent_name,
            update={"messages": state["messages"] + [tool_message]},
            graph=Command.PARENT,
        )
    return handoff_tool

# 旅行顾问代理
travel_advisor_tools = [
    get_travel_recommendations,
    create_handoff_tool(agent_name="hotel_advisor"),
]
travel_advisor = create_react_agent(
    model,
    travel_advisor_tools,
    prompt="You are a general travel expert.",
)

def call_travel_advisor(state: MultiAgentState) -> Command[Literal["human"]]:
    response = travel_advisor.invoke(state)
    update = {**response, "last_active_agent": "travel_advisor"}
    return Command(update=update, goto="human")

# 酒店顾问代理
hotel_advisor_tools = [
    get_hotel_recommendations,
    create_handoff_tool(agent_name="travel_advisor"),
]
hotel_advisor = create_react_agent(
    model,
    hotel_advisor_tools,
    prompt="You are a hotel expert.",
)

def call_hotel_advisor(state: MultiAgentState) -> Command[Literal["human"]]:
    response = hotel_advisor.invoke(state)
    update = {**response, "last_active_agent": "hotel_advisor"}
    return Command(update=update, goto="human")

# 人工节点
def human_node(state: MultiAgentState):
    user_input = interrupt(value="Ready for user input")
    active_agent = state["last_active_agent"]
    
    return Command(
        update={
            "messages": [{"role": "human", "content": user_input}]
        },
        goto=active_agent,
    )

# 构建图
builder = StateGraph(MultiAgentState)
builder.add_node("travel_advisor", call_travel_advisor)
builder.add_node("hotel_advisor", call_hotel_advisor)
builder.add_node("human", human_node)
builder.add_edge(START, "travel_advisor")

checkpointer = MemorySaver()
graph = builder.compile(checkpointer=checkpointer)

# 使用
thread_config = {"configurable": {"thread_id": str(uuid.uuid4())}}

# 多轮对话
inputs = [
    {"messages": [{"role": "user", "content": "Caribbean vacation"}]},
    Command(resume="Recommend a hotel"),
    Command(resume="I like the first option"),
]

for user_input in inputs:
    for update in graph.stream(user_input, config=thread_config, stream_mode="updates"):
        for node_id, value in update.items():
            if value.get("messages"):
                print(f"{node_id}: {value['messages'][-1].content}")
```

## 最佳实践

1. **选择合适的架构**
   - 简单系统：网络或主管
   - 复杂系统：分层或自定义工作流

2. **设计清晰的交接**
   - 明确交接条件
   - 传递足够的上下文
   - 记录交接历史

3. **管理消息历史**
   - 考虑共享 vs 私有消息
   - 实现记忆管理策略
   - 标识消息来源代理

4. **支持多轮对话**
   - 使用 interrupt 收集输入
   - 跟踪活跃代理
   - 正确路由回复

5. **监控和调试**
   - 使用 LangSmith 追踪代理交互
   - 记录代理决策
   - 设置超时和错误处理
