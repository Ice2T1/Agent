# LangGraph 断点与调试

## 概述

断点可以在定义的点暂停图执行，让你逐步检查每个阶段。断点利用 LangGraph 的持久层，在每一步后保存图状态。

## 主要功能

- **暂停执行**：在预定义点无限期暂停
- **检查状态**：查看图的当前状态和节点输入
- **逐步执行**：一次执行一个节点
- **无限期暂停**：由于检查点持久化，暂停时间不受限制

## 使用场景

### 1. 调试复杂工作流

```python
from langgraph.graph import StateGraph, START

# 编译时指定断点
graph = builder.compile(
    checkpointer=checkpointer,
    interrupt_before=["node_name"]  # 在执行 node_name 之前暂停
)

# 或在中断后恢复
config = {"configurable": {"thread_id": "1"}}
result = graph.invoke(input_data, config)

# 检查状态
state = graph.get_state(config)
print(state.values)

# 恢复执行
result = graph.invoke(None, config)
```

### 2. 人工审查

```python
# 在关键节点前暂停，等待人工审查
graph = builder.compile(
    interrupt_before=["critical_api_call"]
)

# 执行到断点
result = graph.invoke(input_data, config)

# 检查即将执行的节点
snapshot = graph.get_state(config)
print(f"Next node: {snapshot.next}")

# 人工审查后恢复
result = graph.invoke(None, config)
```

## 配置断点

### interrupt_before

在指定节点之前中断：

```python
graph = builder.compile(
    interrupt_before=["node_a", "node_b"]
)
```

### interrupt_after

在指定节点之后中断：

```python
graph = builder.compile(
    interrupt_after=["node_a"]
)
```

## 检查执行状态

### 获取当前状态

```python
config = {"configurable": {"thread_id": "1"}}
state_snapshot = graph.get_state(config)

print(f"当前值：{state_snapshot.values}")
print(f"下一个节点：{state_snapshot.next}")
print(f"元数据：{state_snapshot.metadata}")
```

### 获取执行历史

```python
history = list(graph.get_state_history(config))
for snapshot in history:
    print(f"Step: {snapshot.metadata['step']}, Next: {snapshot.next}")
```

## 实际应用示例

### 示例 1：审查工具调用

```python
from langgraph.graph import StateGraph, MessagesState

def call_llm(state: MessagesState):
    # LLM 决定调用哪个工具
    response = llm.invoke(state["messages"])
    return {"messages": [response]}

def execute_tool(state: MessagesState):
    # 执行工具调用
    tool_call = state["messages"][-1].tool_calls[0]
    result = tools[tool_call["name"]].invoke(tool_call["args"])
    return {"messages": [result]}

builder = StateGraph(MessagesState)
builder.add_node("call_llm", call_llm)
builder.add_node("execute_tool", execute_tool)
builder.add_edge(START, "call_llm")
builder.add_edge("call_llm", "execute_tool")

# 在工具执行前暂停，等待审查
graph = builder.compile(
    checkpointer=checkpointer,
    interrupt_before=["execute_tool"]
)

# 使用
config = {"configurable": {"thread_id": "1"}}

# 第一次调用：执行到 execute_tool 之前
result = graph.invoke({"messages": [HumanMessage(content="What's the weather?")]}, config)

# 检查即将执行的工具
state = graph.get_state(config)
print("Pending tool call:", state.values["messages"][-1].tool_calls)

# 人工审查通过后恢复
result = graph.invoke(None, config)
```

### 示例 2：多步审批流程

```python
def draft_content(state: State):
    content = llm.invoke("Draft marketing content")
    return {"content": content, "status": "draft"}

def review_content(state: State):
    # 这里会中断，等待人工审查
    return {"status": "under_review"}

def approve_content(state: State):
    return {"status": "approved"}

builder = StateGraph(State)
builder.add_node("draft", draft_content)
builder.add_node("review", review_content)
builder.add_node("approve", approve_content)
builder.add_edge(START, "draft")
builder.add_edge("draft", "review")
builder.add_edge("review", "approve")

# 在审查节点前中断
graph = builder.compile(
    checkpointer=checkpointer,
    interrupt_before=["review"]
)

# 工作流
config = {"configurable": {"thread_id": "1"}}

# 生成草稿
result = graph.invoke({"topic": "New product"}, config)

# 人工审查草稿
state = graph.get_state(config)
print("Draft content:", state.values["content"])

# 可能需要修改草稿
graph.update_state(config, {"content": "Revised content"})

# 继续执行
result = graph.invoke(None, config)
```

## 与时间旅行结合

断点可以与时间旅行功能结合使用：

```python
# 执行到断点
result = graph.invoke(input_data, config)

# 查看历史
history = graph.get_state_history(config)

# 回滚到之前的检查点
old_checkpoint_id = history[-1].config["configurable"]["checkpoint_id"]
config["configurable"]["checkpoint_id"] = old_checkpoint_id

# 从旧检查点恢复（创建新分支）
result = graph.invoke(None, config)
```

## 常见问题

### Q: 断点会超时吗？

不会。由于检查点持久化，执行可以无限期暂停。

### Q: 如何在暂停时修改状态？

```python
# 获取当前状态
state = graph.get_state(config)

# 更新状态
graph.update_state(config, {"key": "new_value"})

# 恢复执行
graph.invoke(None, config)
```

### Q: 可以设置多个断点吗？

可以。编译时指定多个节点：

```python
graph = builder.compile(
    interrupt_before=["node_a", "node_b", "node_c"]
)
```

### Q: 断点会影响性能吗？

断点本身不影响性能，因为：
- 检查点在每个步骤后都会保存
- 暂停时不消耗计算资源
- 恢复时从检查点继续

## 完整示例

```python
from langgraph.graph import StateGraph, MessagesState, START
from langgraph.checkpoint.memory import InMemorySaver
from langchain_core.messages import HumanMessage, AIMessage
from typing import TypedDict, Optional

class State(TypedDict):
    messages: list
    status: str
    review_feedback: Optional[str]

# 检查点器
checkpointer = InMemorySaver()

# 节点
def generate_response(state: MessagesState):
    response = llm.invoke(state["messages"])
    return {"messages": [response]}

def check_sensitive_content(state: State):
    content = state["messages"][-1].content
    if contains_sensitive_info(content):
        return {"status": "needs_review"}
    return {"status": "approved"}

def human_review(state: State):
    # 这里会中断，等待人工审查
    return state

def apply_feedback(state: State):
    if state.get("review_feedback"):
        # 根据反馈修改
        revised = revise_based_on_feedback(
            state["messages"][-1],
            state["review_feedback"]
        )
        return {"messages": [revised], "status": "approved"}
    return state

# 构建图
builder = StateGraph(State)
builder.add_node("generate", generate_response)
builder.add_node("check", check_sensitive_content)
builder.add_node("review", human_review)
builder.add_node("apply", apply_feedback)

builder.add_edge(START, "generate")
builder.add_edge("generate", "check")
builder.add_conditional_edges(
    "check",
    lambda s: "review" if s["status"] == "needs_review" else "approve",
    {"review": "review", "approve": END}
)
builder.add_edge("review", "apply")
builder.add_edge("apply", END)

# 编译时设置断点
graph = builder.compile(
    checkpointer=checkpointer,
    interrupt_before=["review"]  # 在人工审查前暂停
)

# 使用
config = {"configurable": {"thread_id": "1"}}

# 执行到断点
result = graph.invoke(
    {"messages": [HumanMessage(content="Generate report with API keys")]},
    config
)

# 检查状态
state = graph.get_state(config)
print("需要审查:", state.values)

# 提供反馈
graph.update_state(
    config,
    {"review_feedback": "Remove API keys, use placeholders"}
)

# 恢复执行
result = graph.invoke(None, config)
print("最终结果:", result)
```
