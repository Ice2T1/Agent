# LangGraph 持久化与内存管理

## 持久化层（检查点）

LangGraph 通过检查点器（Checkpointer）实现持久化层。使用检查点器编译图时，检查点器会在每个超级步骤保存图状态的检查点。

### 核心概念

**线程（Thread）**
- 分配给检查点器保存的每个检查点的唯一 ID
- 必须在配置的 `configurable` 部分中指定 `thread_id`

```python
{"configurable": {"thread_id": "1"}}
```

**检查点（Checkpoint）**
- 每个超级步骤保存的图状态快照
- 由 `StateSnapshot` 对象表示，包含：
  - `config`：与此检查点关联的配置
  - `metadata`：元数据
  - `values`：状态通道的值
  - `next`：接下来要执行的节点名称元组
  - `tasks`：后续任务信息

### 使用检查点器

```python
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import InMemorySaver
from typing import Annotated
from typing_extensions import TypedDict
from operator import add

class State(TypedDict):
    foo: str
    bar: Annotated[list[str], add]

def node_a(state: State):
    return {"foo": "a", "bar": ["a"]}

def node_b(state: State):
    return {"foo": "b", "bar": ["b"]}

workflow = StateGraph(State)
workflow.add_node(node_a)
workflow.add_node(node_b)
workflow.add_edge(START, "node_a")
workflow.add_edge("node_a", "node_b")
workflow.add_edge("node_b", END)

checkpointer = InMemorySaver()
graph = workflow.compile(checkpointer=checkpointer)

config = {"configurable": {"thread_id": "1"}}
graph.invoke({"foo": ""}, config)
```

### 获取状态

**获取最新状态**

```python
config = {"configurable": {"thread_id": "1"}}
state = graph.get_state(config)
```

**获取特定检查点的状态**

```python
config = {
    "configurable": {
        "thread_id": "1",
        "checkpoint_id": "1ef663ba-28fe-6528-8002-5a559208592c"
    }
}
state = graph.get_state(config)
```

**StateSnapshot 示例**

```python
StateSnapshot(
    values={'foo': 'b', 'bar': ['a', 'b']},
    next=(),
    config={'configurable': {'thread_id': '1', ...}},
    metadata={'source': 'loop', 'writes': {'node_b': {...}}, 'step': 2},
    created_at='2024-08-29T19:19:38.821749+00:00',
    parent_config={'configurable': {...}},
    tasks=()
)
```

### 获取状态历史

```python
config = {"configurable": {"thread_id": "1"}}
history = list(graph.get_state_history(config))
```

检查点按时间顺序排列，最近的在最前面。

### 回放执行

从特定检查点回放：

```python
config = {
    "configurable": {
        "thread_id": "1",
        "checkpoint_id": "0c62ca34-ac19-445d-bbb0-5b4984975b2a"
    }
}
graph.invoke(None, config=config)
```

- LangGraph 会回放该检查点之前已执行的步骤（不重新执行）
- 检查点之后的步骤将被执行（新分支）

### 更新状态

```python
graph.update_state(config, values, as_node=None)
```

**参数**：
- `config`：包含 `thread_id`，可选 `checkpoint_id`
- `values`：用于更新状态的值
- `as_node`：可选，指定更新像来自哪个节点

**更新行为**：
- 更新的处理方式与来自节点的更新完全相同
- 如果通道定义了 reducer 函数，值将传递给 reducer

```python
# 当前状态
{"foo": 1, "bar": ["a"]}

# 更新
graph.update_state(config, {"foo": 2, "bar": ["b"]})

# 新状态
{"foo": 2, "bar": ["a", "b"]}  # foo 被覆盖，bar 被累加
```

## 记忆存储（Store）

检查点器实现**线程内**持久化，而 `Store` 接口支持**跨线程**共享信息。

### 基本用法

```python
from langgraph.store.memory import InMemoryStore
import uuid

in_memory_store = InMemoryStore()

# 命名空间（类似文件夹）
user_id = "1"
namespace = (user_id, "memories")

# 保存记忆
memory_id = str(uuid.uuid4())
memory = {"food_preference": "I like pizza"}
in_memory_store.put(namespace, memory_id, memory)

# 读取记忆
memories = in_memory_store.search(namespace)
latest_memory = memories[-1].dict()
# {'value': {'food_preference': 'I like pizza'},
#  'key': '07e0caf4-1631-47b7-b15f-65515d4c1843',
#  'namespace': ['1', 'memories'],
#  'created_at': '2024-10-02T17:22:31.590602+00:00',
#  'updated_at': '2024-10-02T17:22:31.590605+00:00'}
```

### Item 属性

- `value`：记忆的值（字典）
- `key`：记忆的唯一键
- `namespace`：命名空间（字符串列表）
- `created_at`：创建时间戳
- `updated_at`：更新时间戳

### 语义搜索

使用嵌入模型配置存储：

```python
from langchain.embeddings import init_embeddings

store = InMemoryStore(
    index={
        "embed": init_embeddings("openai:text-embedding-3-small"),
        "dims": 1536,
        "fields": ["food_preference", "$"]
    }
)

# 自然语言查询
memories = store.search(
    namespace,
    query="What does the user like to eat?",
    limit=3
)
```

**存储时指定嵌入字段**：

```python
# 只嵌入特定字段
store.put(
    namespace,
    str(uuid.uuid4()),
    {
        "food_preference": "I love Italian cuisine",
        "context": "Discussing dinner plans"
    },
    index=["food_preference"]
)

# 不嵌入（可检索但不可搜索）
store.put(
    namespace,
    str(uuid.uuid4()),
    {"system_info": "Last updated: 2024-01-01"},
    index=False
)
```

## 检查点库

### 可用实现

**langgraph-checkpoint**
- 基础接口：`BaseCheckpointSaver` 和 `SerializerProtocol`
- 包含 `InMemorySaver`（用于实验）
- LangGraph 已包含此库

**langgraph-checkpoint-sqlite**（需单独安装）
- 使用 SQLite 数据库：`SqliteSaver` / `AsyncSqliteSaver`
- 适合实验和本地工作流

**langgraph-checkpoint-postgres**（需单独安装）
- 使用 Postgres 数据库：`PostgresSaver` / `AsyncPostgresSaver`
- 适合生产环境（LangGraph 平台使用）

### 检查点接口

所有检查点器都实现 `BaseCheckpointSaver` 接口：

```python
# 核心方法
.put - 存储带配置和元数据的检查点
.put_writes - 存储中间写入（待处理写入）
.get_tuple - 使用配置获取检查点元组
.list - 列出匹配的检查点

# 异步版本（用于异步图执行）
.aput
.aput_writes
.aget_tuple
.alist
```

### 序列化器

检查点器使用序列化器序列化状态通道值：

- 默认：`JsonPlusSerializer`
- 处理各种类型：LangChain/LangGraph 类型、日期时间、枚举等

### 加密

```python
from langgraph.checkpoint.serde.encrypted import EncryptedSerializer
from langgraph.checkpoint.sqlite import SqliteSaver
import sqlite3

serde = EncryptedSerializer.from_pycryptodome_aes()  # 读取 LANGGRAPH_AES_KEY
checkpointer = SqliteSaver(sqlite3.connect("checkpoint.db"), serde=serde)
```

## 内存管理

### 短期记忆（线程范围）

短期记忆作为代理状态的一部分管理，通过检查点持久化。

#### 管理长对话历史

**问题**：
- 完整历史可能超出 LLM 上下文窗口
- 长上下文下 LLM 表现不佳
- 成本高、响应慢

**解决方案 1：编辑消息列表**

```python
from langchain_core.messages import RemoveMessage, AIMessage
from langgraph.graph import add_messages

class State(TypedDict):
    messages: Annotated[list, add_messages]

def my_node_1(state: State):
    return {"messages": [AIMessage(content="Hi")]}

def my_node_2(state: State):
    # 删除除最后 2 条外的所有消息
    delete_messages = [RemoveMessage(id=m.id) for m in state['messages'][:-2]]
    return {"messages": delete_messages}
```

`add_messages` reducer：
- 追加新消息到列表
- 看到 `RemoveMessage` 时，从列表中删除该 ID 的消息

**解决方案 2：总结对话**

```python
class State(MessagesState):
    summary: str

def summarize_conversation(state: State):
    summary = state.get("summary", "")
    
    if summary:
        summary_message = f"This is a summary: {summary}\n\nExtend it:"
    else:
        summary_message = "Create a summary:"
    
    messages = state["messages"] + [HumanMessage(content=summary_message)]
    response = model.invoke(messages)
    
    # 删除除最后 2 条外的所有消息
    delete_messages = [RemoveMessage(id=m.id) for m in state["messages"][:-2]]
    return {"summary": response.content, "messages": delete_messages}
```

**使用 trim_messages 工具**：

```python
from langchain_core.messages import trim_messages

trimmed = trim_messages(
    messages,
    strategy="last",
    token_counter=ChatOpenAI(model="gpt-4"),
    max_tokens=45,
    start_on="human",
    end_on=("human", "tool"),
    include_system=True,
)
```

### 长期记忆（跨线程）

长期记忆保存在自定义命名空间中，跨线程共享。

#### 记忆类型

| 类型 | 存储内容 | 代理示例 |
|------|---------|---------|
| **语义记忆** | 事实 | 关于用户的事实、偏好 |
| **情景记忆** | 经验 | 代理过去的行动、少样本示例 |
| **程序记忆** | 指令 | 代理系统提示、规则 |

#### 语义记忆实现

**个人资料方式**（单一文档）：

```python
namespace = (user_id, "profile")
store.put(
    namespace,
    "profile",
    {
        "preferences": ["likes short language", "prefers Python"],
        "history": "Long-time user"
    }
)
```

**集合方式**（多个文档）：

```python
namespace = (user_id, "memories")
store.put(
    namespace,
    str(uuid.uuid4()),
    {"fact": "User likes Italian food", "confidence": 0.9}
)
store.put(
    namespace,
    str(uuid.uuid4()),
    {"fact": "User works as a developer", "confidence": 0.95}
)
```

#### 情景记忆（少样本示例）

使用 LangSmith 数据集存储少样本示例：

    namespace = ("agent_instructions",)
    instructions = store.get(namespace, key="agent_a")[0]
    prompt = prompt_template.format(instructions=instructions.value["instructions"])
    # ...

# 更新指令的节点
def update_instructions(state: State, store: BaseStore):
    namespace = ("instructions",)
    current_instructions = store.search(namespace)[0]
    
    prompt = prompt_template.format(
        instructions=current_instructions.value["instructions"],
        conversation=state["messages"]

    output = llm.invoke(prompt)
    new_instructions = output['new_instructions']
    
    store.put(("agent_instructions",), "agent_a", {"instructions": new_instructions})

#### 写入记忆的时机

**热路径写入**（运行时）
- ✅ 实时更新，新记忆立即可用
- ✅ 透明，可通知用户
- ❌ 增加延迟
- ❌ 代理需多任务处理

**后台写入**（异步）
- ✅ 不影响主流程延迟
- ✅ 可批量处理
- ❌ 新记忆不立即可用
- ❌ 需要额外的后台处理逻辑

## 最佳实践

1. **选择合适的检查点器**
   - 实验：`InMemorySaver`
   - 本地开发：`SqliteSaver`
   - 生产：`PostgresSaver`

2. **管理对话历史**
   - 短对话：使用 `RemoveMessage`
   - 长对话：结合总结和修剪

3. **设计记忆策略**
   - 明确记忆类型（语义/情景/程序）
   - 选择合适的存储方式（个人资料 vs 集合）
   - 配置语义搜索嵌入

4. **性能优化**
   - 限制记忆搜索数量
   - 使用命名空间隔离
   - 定期清理过期记忆
