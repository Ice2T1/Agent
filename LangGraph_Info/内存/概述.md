# LangGraph 内存管理指南

## 什么是内存？

内存使 AI 代理能够存储、检索和使用信息来理解现在和未来。对于承担复杂任务的代理，配备内存对于效率和用户满意度至关重要。

## 两种内存类型

### 1. 短期记忆（线程范围）

- **范围**：单个用户对话线程
- **管理方式**：作为代理状态的一部分
- **持久化**：通过检查点保存到数据库
- **更新时机**：图被调用或步骤完成时
- **读取时机**：每个步骤开始时

### 2. 长期记忆（跨线程）

- **范围**：对话线程之间共享
- **管理方式**：存储在自定义命名空间
- **持久化**：通过 Store 接口
- **访问时机**：任何时候、任何线程

## 短期记忆管理

### 挑战：长对话历史

**问题**：
- 完整历史超出 LLM 上下文窗口
- 长上下文下 LLM 表现不佳（被过时内容"分散注意力"）
- 响应时间慢、成本高

### 技术方案

#### 方案 1：编辑消息列表

**方法 A：自定义 reducer**

```python
from typing import Union
from typing_extensions import TypedDict
from typing import Annotated

def manage_list(existing: list, updates: Union[list, dict]):
    if isinstance(updates, list):
        return existing + updates
    elif isinstance(updates, dict) and updates["type"] == "keep":
        return existing[updates["from"]:updates["to"]]
    # 可自定义其他逻辑

class State(TypedDict):
    my_list: Annotated[list, manage_list]

def my_node(state: State):
    return {
        # 只保留最后 5 条消息
        "my_list": {"type": "keep", "from": -5, "to": None}
    }
```

**方法 B：使用 RemoveMessage**

```python
from langchain_core.messages import RemoveMessage, AIMessage
from langgraph.graph import add_messages

class State(TypedDict):
    messages: Annotated[list, add_messages]

def my_node_1(state: State):
    # 添加消息
    return {"messages": [AIMessage(content="Hi")]}

def my_node_2(state: State):
    # 删除除最后 2 条外的所有消息
    delete_messages = [RemoveMessage(id=m.id) for m in state['messages'][:-2]]
    return {"messages": delete_messages}
```

`add_messages` reducer 行为：
- 正常消息：追加到现有列表
- `RemoveMessage`：从列表中删除该 ID 的消息

#### 方案 2：总结对话

```python
from langgraph.graph import MessagesState
from langchain_core.messages import HumanMessage, RemoveMessage

class State(MessagesState):
    summary: str

def summarize_conversation(state: State):
    # 获取现有摘要
    summary = state.get("summary", "")
    
    # 创建摘要提示
    if summary:
        summary_message = (
            f"This is a summary of the conversation to date: {summary}\n\n"
            "Extend the summary by taking into account the new messages above:"
        )
    else:
        summary_message = "Create a summary of the conversation above:"
    
    # 添加到历史
    messages = state["messages"] + [HumanMessage(content=summary_message)]
    response = model.invoke(messages)
    
    # 删除除最后 2 条外的所有消息
    delete_messages = [RemoveMessage(id=m.id) for m in state["messages"][:-2]]
    return {"summary": response.content, "messages": delete_messages}
```

#### 方案 3：使用 trim_messages 工具

```python
from langchain_core.messages import trim_messages
from langchain_openai import ChatOpenAI

trimmed = trim_messages(
    messages,
    # 保留策略："last" 保留最后 n 个 token
    strategy="last",
    # token 计数器（根据模型调整）
    token_counter=ChatOpenAI(model="gpt-4"),
    # 最大 token 数
    max_tokens=45,
    # 大多数模型要求以 human 消息开始
    start_on="human",
    # 以 human 或 tool 消息结束
    end_on=("human", "tool"),
    # 保留 system 消息
    include_system=True,
)
```

**注意事项**：
- 某些模型 API 限制消息类型顺序
- 需要计算 token 数量来决定何时截断

## 长期记忆管理

### 存储结构

LangGraph 将长期记忆作为 JSON 文档存储在 Store 中：

```python
from langgraph.store.memory import InMemoryStore

# 嵌入函数（替换为实际实现）
def embed(texts: list[str]) -> list[list[float]]:
    return [[1.0, 2.0] * len(texts)]

store = InMemoryStore(index={"embed": embed, "dims": 2})

# 命名空间（类似文件夹路径）
user_id = "my-user"
application_context = "chitchat"
namespace = (user_id, application_context)

# 保存记忆
store.put(
    namespace,
    "a-memory",  # 唯一 key
    {
        "rules": [
            "User likes short, direct language",
            "User only speaks English & python",
        ],
        "my-key": "my-value",
    },
)

# 通过 ID 获取
item = store.get(namespace, "a-memory")

# 搜索（支持过滤和语义搜索）
items = store.search(
    namespace,
    filter={"my-key": "my-value"},
    query="language preferences"
)
```

### 记忆类型框架

设计长期记忆时考虑以下问题：

1. **记忆的类型是什么？**
   - 语义记忆（事实）
   - 情景记忆（经验）
   - 程序记忆（指令）

2. **何时更新记忆？**
   - 热路径（运行时，实时但增加延迟）
   - 后台（异步，不阻塞但非实时）

### 语义记忆（Semantic Memory）

存储事实和概念，用于个性化应用。

#### 实现方式 A：个人资料（单一文档）

```python
namespace = (user_id, "profile")

# 更新配置文件（需要传入先前的配置文件）
store.put(
    namespace,
    "profile",
    {
        "preferences": ["likes short language", "prefers Python"],
        "history": "Long-time user",
        "last_updated": "2024-01-01"
    }
)
```

**优点**：
- 结构简单
- 易于管理
- 完整上下文

**挑战**：
- 配置文件变大时容易出错
- 需要协调新旧信息
- 可能需要拆分为多个文档

#### 实现方式 B：集合（多个文档）

```python
namespace = (user_id, "memories")

# 创建新记忆
store.put(
    namespace,
    str(uuid.uuid4()),
    {"fact": "User likes Italian food", "confidence": 0.9, "source": "conversation"}
)

store.put(
    namespace,
    str(uuid.uuid4()),
    {"fact": "User works as a developer", "confidence": 0.95}
)
```

**优点**：
- 每个记忆范围狭窄，易于生成
- 更高的下游召回率
- 模型更容易生成新对象

**挑战**：
- 需要管理插入/更新/删除
- 模型可能默认过度插入或过度更新
- 记忆之间缺乏完整上下文
- 搜索复杂性增加

**建议**：使用 Trustcall 包管理更新行为，配合评估工具（如 LangSmith）调整行为。

### 情景记忆（Episodic Memory）

回忆过去的事件或行动，通常用于少样本提示。

#### 使用 LangSmith 数据集

```python
from langsmith import Client

client = Client()

# 检索少样本示例
examples = client.list_examples(
    dataset_name="my_fewshot_examples",
    metadata={"tags": ["tool_calling"]}
)

# 动态选择最相关的示例
# （LangSmith 支持基于 BM25 的关键词相似性检索）
```

**优点**：
- "展示"比"告知"更容易
- LLM 从示例中学得很好
- 可紧密集成评估工具

**挑战**：
- 需要选择最相关的示例
- 存储和管理大量示例

### 程序记忆（Procedural Memory）

记住执行任务的规则，是模型权重、代码和提示的组合。

#### 更新代理指令

```python
# 使用记忆的节点
def call_model(state: State, store: BaseStore):
    namespace = ("agent_instructions",)
    instructions = store.get(namespace, key="agent_a")[0]
    
    prompt = prompt_template.format(
        instructions=instructions.value["instructions"]
    )
    # ...

# 更新指令的节点
def update_instructions(state: State, store: BaseStore):
    namespace = ("instructions",)
    current_instructions = store.search(namespace)[0]
    
    # 基于当前指令和对话反馈更新指令
    prompt = prompt_template.format(
        instructions=current_instructions.value["instructions"],
        conversation=state["messages"]
    )
    output = llm.invoke(prompt)
    new_instructions = output['new_instructions']
    
    store.put(
        ("agent_instructions",),
        "agent_a",
        {"instructions": new_instructions}
    )
```

**应用场景**：
- 难以预先指定指令的任务
- 需要从交互中学习和适应
- 基于外部反馈完善提示

## 写入记忆的时机

### 热路径写入（运行时）

```python
def agent_node(state: State, store: BaseStore):
    # 分析对话
    memory_to_save = extract_memory(state["messages"])
    
    # 实时保存记忆
    store.put(namespace, str(uuid.uuid4()), memory_to_save)
    
    # 继续代理逻辑
    response = model.invoke(...)
    return {"messages": [response]}
```

**优点**：
- ✅ 实时更新，新记忆立即可用
- ✅ 透明，可在创建时通知用户
- ✅ 简单直接

**缺点**：
- ❌ 增加延迟
- ❌ 代理需多任务处理（对话 + 记忆管理）
- ❌ 可能影响记忆质量

### 后台写入（异步）

```python
# 主代理流程
def agent_node(state: State):
    response = model.invoke(...)
    return {"messages": [response], "pending_memories": [...]}

# 后台任务
def memory_writer(state: State, store: BaseStore):
    for memory in state.get("pending_memories", []):
        store.put(namespace, str(uuid.uuid4()), memory)
```

**优点**：
- ✅ 不影响主流程延迟
- ✅ 可批量处理
- ✅ 代理专注于主要任务

**缺点**：
- ❌ 新记忆不立即可用
- ❌ 需要额外的后台处理逻辑
- ❌ 复杂性增加

## 最佳实践

### 1. 选择合适的记忆策略

- **短对话**：短期记忆（消息历史）
- **长期个性化**：语义记忆（个人资料或集合）
- **任务学习**：情景记忆（少样本示例）
- **自适应行为**：程序记忆（更新指令）

### 2. 管理记忆质量

- 为记忆添加置信度分数
- 定期清理过期或矛盾的记忆
- 使用评估工具监控记忆效果

### 3. 优化性能

- 限制搜索返回的记忆数量
- 使用命名空间隔离不同类型的记忆
- 为常用记忆设置缓存

### 4. 设计记忆模式

- 保持模式简单且一致
- 为记忆添加元数据（时间戳、来源等）
- 考虑记忆的生命周期管理

### 5. 测试与评估

- 使用 LangSmith 等工具评估记忆效果
- 测试不同记忆策略的影响
- 收集用户反馈优化记忆管理

```python
from langgraph.graph import StateGraph, MessagesState
from langgraph.checkpoint.memory import InMemorySaver
from langgraph.store.memory import InMemoryStore
from langchain_core.messages import HumanMessage, AIMessage, RemoveMessage
from typing import Annotated
import uuid

# 状态定义
class State(MessagesState):
    summary: str

# 初始化存储
checkpointer = InMemorySaver()
store = InMemoryStore(index={
    "embed": your_embedding_function,
    "dims": 1536,
    "fields": ["$"]
})

# 编译图
graph = StateGraph(State).compile(
    checkpointer=checkpointer,
    store=store
)

# 节点实现
def chat_node(state: State, config: RunnableConfig, store: BaseStore):
    user_id = config["configurable"]["user_id"]
    namespace = (user_id, "memories")
    
    # 检索相关记忆
    memories = store.search(
        namespace,
        query=state["messages"][-1].content,
        limit=3
    )
    context = "\n".join([m.value["fact"] for m in memories])
    
    # 使用记忆生成响应
    prompt = f"Context: {context}\n\nUser: {state['messages'][-1].content}"
    response = model.invoke(prompt)
    
    return {"messages": [response]}

# 运行
```
