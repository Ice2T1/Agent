# LangGraph 流式传输

## 概述

LangGraph 的流式传输系统提供实时更新，实现响应迅速且透明的用户体验。

## 可流式传输的数据类型

### 1. 工作流进度
- 在每个图节点执行后获取状态更新
- 适合显示处理进度

### 2. LLM 令牌
- 在语言模型令牌生成时实时流式传输
- 适合逐字显示 AI 响应

### 3. 自定义更新
- 发出用户定义的信号
- 例如："已获取 10/100 条记录"

## 主要功能

- ✅ **流式传输 LLM 令牌**：从节点内、子图或工具中捕获令牌流
- ✅ **从工具发出进度通知**：直接从工具函数发送自定义更新
- ✅ **从子图流式传输**：包括父图和嵌套子图的输出
- ✅ **使用任何 LLM**：即使不是 LangChain 模型也可流式传输
- ✅ **多种流式传输模式**：values、updates、messages、custom、debug

## 流式传输模式

### 1. updates（默认）

流式传输状态增量更新：

```python
for update in graph.stream(input_data, stream_mode="updates"):
    for node_name, node_update in update.items():
        print(f"Node {node_name} updated: {node_update}")
```

**特点**：
- 只显示变化的部分
- 适合跟踪哪些节点执行了
- 数据量小

### 2. values

流式传输完整状态：

```python
for state in graph.stream(input_data, stream_mode="values"):
    print(f"Current state: {state}")
```

**特点**：
- 显示完整状态
- 适合需要完整上下文的场景
- 数据量较大

### 3. messages

流式传输 LLM 令牌和元数据：

```python
for chunk in graph.stream(input_data, stream_mode="messages"):
    print(chunk.content, end="", flush=True)
```

**特点**：
- 逐字显示 LLM 响应
- 包含消息元数据
- 适合聊天界面

### 4. custom

流式传输用户自定义数据：

```python
def my_node(state, config, writer):
    # 发送自定义更新
    writer({"type": "progress", "message": "Processing..."})
    # 节点逻辑
    return {"result": "done"}

for chunk in graph.stream(input_data, stream_mode="custom"):
    print(f"Custom update: {chunk}")
```

**特点**：
- 完全自定义
- 适合发送进度、日志等
- 需要手动实现

### 5. debug

流式传输详细跟踪信息：

```python
for chunk in graph.stream(input_data, stream_mode="debug"):
    print(f"Debug: {chunk}")
```

**特点**：
- 包含执行详情
- 适合调试
- 数据量最大

```python
from langgraph.graph import StateGraph

graph = builder.compile()
```

### 示例 3：从工具发送进度

```python
from langgraph.types import StreamWriter

def search_tool(query: str, writer: StreamWriter):
    writer({"type": "progress", "message": f"Searching for: {query}"})
    
    results = []
    for i, result in enumerate(search_api(query)):
        results.append(result)
        writer({
            "type": "progress",
            "message": f"Found {len(results)} results so far..."
        })
    
    return {"results": results}

def process_node(state: State, writer: StreamWriter):
    writer({"type": "status", "message": "Starting processing"})
    
    result = search_tool(state["query"], writer)
    
    writer({"type": "status", "message": "Processing complete"})
    return result

graph = builder.compile()

for update in graph.stream(
    {"query": "AI news"},
    stream_mode="custom"
):
    if update["type"] == "progress":
        print(f"Progress: {update['message']}")
    elif update["type"] == "status":
        print(f"Status: {update['message']}")
```

### 示例 4：子图流式传输

```python
# 子图
def subgraph_node_1(state):
    return {"sub_result": "from sub 1"}

def subgraph_node_2(state):
    return {"sub_result": "from sub 2"}

subgraph = (
    StateGraph(SubState)
    .add_node(subgraph_node_1)
    .add_node(subgraph_node_2)
    .add_edge(START, "subgraph_node_1")
    .add_edge("subgraph_node_1", "subgraph_node_2")
    .compile()
)

# 父图
def parent_node(state):
    return {"parent_result": "done"}

parent = (
    StateGraph(ParentState)
    .add_node("subgraph", subgraph)  # 子图作为节点
    .add_node("parent", parent_node)
    .add_edge(START, "subgraph")
    .add_edge("subgraph", "parent")
    .compile()
)

# 流式传输时设置 subgraphs=True
for chunk in parent.stream(
    input_data,
    stream_mode="updates",
    subgraphs=True  # 包含子图输出
):
    # chunk 是 (namespace, update) 元组
    if isinstance(chunk, tuple):
        namespace, update = chunk
        print(f"Subgraph {'.'.join(namespace)}: {update}")
    else:
        print(f"Parent: {chunk}")
```

### 示例 5：多种模式组合

```python
# 同时流式传输多种类型
for chunk in graph.stream(
    input_data,
    stream_mode=["messages", "updates"]  # 多模式
):
    # chunk 是 (mode, data) 元组
    mode, data = chunk
    
    if mode == "messages":
        print(f"Token: {data.content}", end="")
    elif mode == "updates":
        for node_name, value in data.items():
            print(f"\n✓ {node_name} completed")
```

### 示例 6：非 LangChain LLM 的流式传输

```python
def custom_llm_node(state):
    # 使用自定义 LLM
    import requests
    
    response = requests.post(
        "https://api.example.com/generate",
        json={"prompt": state["prompt"]},
        stream=True
    )
    
    # 手动创建令牌流
    for line in response.iter_lines():

            token = line.decode()
            # 使用 custom 模式发送

    return {"complete": True}
```

## 最佳实践

### 1. 选择合适的模式

- **聊天应用**：`messages` 模式
- **进度显示**：`custom` 模式
- **调试**：`debug` 模式
- **一般用途**：`updates` 模式（默认）

### 2. 性能考虑

```python
# 避免发送过多更新
def efficient_node(state, writer):
    # 不好：发送太多更新
    for item in large_list:
        writer({"item": item})  # 可能过多
    
    # 好：批量发送
    batch = []
    for i, item in enumerate(large_list):
        batch.append(item)
        if i % 100 == 0:  # 每 100 条发送一次
            writer({"batch": batch})
            batch = []
    
    return {"complete": True}
```

## 完整示例：聊天机器人

```python
from langgraph.graph import StateGraph, MessagesState, START
from langchain_core.messages import AIMessageChunk, HumanMessage

def llm_node(state: MessagesState):
    # 流式传输 LLM 响应
    messages = state["messages"]
    
    # 使用流式 API
    stream = llm.stream(messages)
    
    # 收集完整响应
    full_response = ""
    for chunk in stream:
        full_response += chunk.content
        # 流式传输会自动处理
    
    return {"messages": [AIMessage(content=full_response)]}

def response_formatter(state: MessagesState):
    # 格式化响应
    response = state["messages"][-1].content
    formatted = format_response(response)
    return {"messages": [AIMessage(content=formatted)]}

# 构建图
builder = StateGraph(MessagesState)
builder.add_node("llm", llm_node)
builder.add_node("format", response_formatter)
builder.add_edge(START, "llm")
builder.add_edge("llm", "format")

graph = builder.compile()

# 使用
async def chat():
    messages = [HumanMessage(content="Explain quantum physics")]
    
    async for chunk in graph.astream(
        {"messages": messages},
        stream_mode="messages"
    ):
        # chunk 是 AIMessageChunk
        print(chunk.content, end="", flush=True)
    
    print()  # 换行
```

## 故障排除

### Q: 流式传输不工作？

检查：
1. LLM 是否支持流式 API（使用 `.stream()` 而非 `.invoke()`）
2. 是否正确设置了 `stream_mode`
3. 节点是否返回了正确的数据类型

### Q: 如何调试流式传输？

```python
# 使用 debug 模式查看详细流程
for chunk in graph.stream(input, stream_mode="debug"):
    print(f"Time: {chunk.get('timestamp')}")
    print(f"Event: {chunk.get('event')}")
    print(f"Data: {chunk.get('data')}")
```

### Q: 如何处理子图流式传输？

确保设置 `subgraphs=True`：

```python
for chunk in graph.stream(input, stream_mode="updates", subgraphs=True):
    # 处理父图和子图更新
```
